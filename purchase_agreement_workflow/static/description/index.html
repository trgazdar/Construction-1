Overview

Purchase >> order >> Purchase Agreements Note: you must active the Purchase Agreements function in purchase settings
1- Purchase Agreements Workflow
a. Stage
i. Draft
ii. Dep. Manager Approval
iii. Approved
iv. Cancel

b. Action
i. Draft: When requester create the Purchase Agreements and save then the Purchase Agreements will be in Draft Stage.
1. Appear new button named confirm after the user click in this button the Purchase Agreements will move to Dep. Manager Approval Stage and send Notification to the user in the manager field in Department form related by the requester
2. Appear New button named Cancel after the user click in this button the Purchase Agreements will move to Cancel Stage

ii.. Dep. Manager Approval
1. Appear new button named Approve for user in the manager field in Department form related by the requester. after the user click in this button the Purchase Agreements will move to Approved Stage and send Notification to all user in that Responsible field in Kind Of Purchase related by that purchase agreements
2. Appear New button named Reject for user in the manager field in Department form related by the requester. after the user click in this button the Purchase Agreements will move to Draft staged and send notification to requester
v. Approved
Appear in this stage the New QUTATION , CLOSE , CANCEL button it is already
